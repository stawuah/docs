## What is swiftui-rn?

**swiftui-rn** lets you build React Native screens the way SwiftUI developers expect — no JSX soup, no `StyleSheet.create()`, no CSS. Every UI element is a **node**. Every style is a **modifier**. You chain them. It renders.

## Why swiftui-rn?

The problem with React Native styling has always been the gap between how you **think** about UI and how you **write** it. You picture a card with padding, a rounded corner, and a shadow — but you end up writing a deeply nested object inside JSX.

swiftui-rn closes that gap.

<CodeGroup>

```tsx Without swiftui-rn
<View style={{
flexDirection: 'column',
padding: 24,
backgroundColor: '#0f172a',
borderRadius: 16,
gap: 12
}}>
<Text style={{
fontSize: 28,
fontWeight: 'bold',
color: '#fff'
}}>
SwiftPay
</Text>
<TouchableOpacity
style={{
backgroundColor: '#16a34a',
padding: 18,
borderRadius: 14
}}
onPress={handleSend}
>
<Text style={{ color: '#fff', fontWeight: '600' }}>
Send Now
</Text>
</TouchableOpacity>
</View>
```


```tsx With swiftui-rn
import { VStack, Text, Button, Render } from 'swiftui-rn'

const screen = VStack(
Text('SwiftPay')
.fontSize(28)
.bold()
.color('#fff'),

Button('Send Now', handleSend)
.bg('#16a34a')
.padding(18)
.rounded(14),
)
.bg('#0f172a')
.padding(24)
.rounded(16)
.gap(12)

export default () => <Render node={screen} />
```

</CodeGroup>

Same output. Same performance. Half the lines. Zero StyleSheet.

## Core concepts

<AccordionGroup>
  <Accordion title="Everything is a node">
    Every UI element — a stack, a text label, a button, an image — is a `ViewNode` object. Nodes hold their style props and children as plain JavaScript data.
  </Accordion>
  <Accordion title="Modifiers are chainable methods">
    Instead of passing a style object, you call methods on the node. Each method returns the same node, so you can chain as many as you want. `.padding(16).rounded(12).shadow()` reads exactly like SwiftUI modifiers.
  </Accordion>
  <Accordion title="One Render component">
    You build your entire screen as a node tree, then pass it to `<Render node={...} />`. The renderer walks the tree and outputs the correct React Native components — `View`, `Text`, `TouchableOpacity`, `Image`, `TextInput`, etc.
  </Accordion>
  <Accordion title="No runtime magic">
    swiftui-rn is a thin wrapper. Node trees are plain objects. `toRNStyle()` is a simple property map. There is no proxy, no compiler, no Babel plugin. What you write is what runs.
  </Accordion>
</AccordionGroup>

## Performance

swiftui-rn adds a wrapper layer on top of React Native — but how much does that cost?

At **~1–5 microseconds per render**, the overhead is imperceptible. React Native's own JS bridge and reconciler cost 10–100x more than the fluent wrapper itself.

<Tip>
  Build static node trees **outside** your component body. Only use `useMemo()` when the tree depends on props or state. This keeps re-render cost at exactly zero.
</Tip>
